Andromeda

A self-hosted, fully offline recovery code vault with end-to-end encryption. No cloud. No tracking. Just yours.

Open Vault → GitHub

Features

Everything you need to store recovery codes safely — and nothing you don't.

🔒

AES-256-GCM Encryption

All vault data is encrypted in your browser using the Web Crypto API before anything touches the server. PBKDF2-SHA256 with 310,000 iterations derives your key. The server stores only an opaque blob.

🌐

Fully Offline After Setup

Zero outbound network calls. No telemetry, no analytics, no CDN dependencies after initial load. DNS resolves to your machine only.

🎨

500+ Service Icons

Built-in icon library covering Proton, Ente, Google, GitHub, AWS, Cloudflare, Bitwarden, Discord and hundreds more — all locally served as emoji or SVG.

🐳

Docker Native

Single container. Runs on amd64 and arm64 (Raspberry Pi, Apple Silicon). One docker run or docker compose up.

✓

Usage Tracking

Click any code to mark it as used. Struck-through, dimmed, always visible. Remaining count shown on each card.

📦

Export Encrypted Backup

Download your vault as a .json anytime. Re-import on another Andromeda instance. The backup is still encrypted — your password is never exported.

🖥

Liquid Glass UI

Pure black-and-white iOS-inspired interface with mouse-reactive glass panels, animated star field, and a custom cursor. No dependencies — no React, no Vue.

Security Model

The encryption happens entirely in your browser. The server is a dumb store.

LayerMethodNotes
EncryptionAES-256-GCMAuthenticated encryption with 128-bit auth tag
Key derivationPBKDF2-SHA256310,000 iterations, 256-bit random salt per save
IV96-bit randomGenerated fresh every save via getRandomValues
StorageSingle blobServer stores only ciphertext — useless without password
TransportHTTPS (recommended)Use Caddy or Traefik in front for TLS termination
⚠
No password recovery. If you lose your master password, your data is gone. There is no backdoor. Keep your password somewhere safe.

For Image Hosters

This section is for you — the person building and publishing the Docker image for others to use.

① Prerequisites

🐳

Docker + Docker Buildx

Install Docker Desktop or Docker Engine 24+. Buildx is bundled in Docker Desktop; for CLI installs run docker buildx install.

🐙

GitHub account

You'll push the image to GitHub Container Registry (GHCR). Create a repo and a Personal Access Token with write:packages scope.

🖥

Node.js 20+ (optional)

Only needed if you want to test locally before building the image.

② Clone & Build the Image

bash
# Clone the repo
git clone https://github.com/yourusername/andromeda.git
cd andromeda

# Build multi-arch image (amd64 + arm64)
docker buildx create --use --name andromeda-builder
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --tag ghcr.io/YOUR_GITHUB_USERNAME/andromeda:latest \
  --tag ghcr.io/YOUR_GITHUB_USERNAME/andromeda:1.0.0 \
  --load \
  .

# Test locally first
docker run -p 3000:3000 ghcr.io/YOUR_GITHUB_USERNAME/andromeda:latest
ℹ
Replace YOUR_GITHUB_USERNAME with your actual GitHub username throughout.

③ Publish to GitHub Container Registry

bash
# Log in to GHCR
echo YOUR_GITHUB_PAT | docker login ghcr.io -u YOUR_GITHUB_USERNAME --password-stdin

# Push the image
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --tag ghcr.io/YOUR_GITHUB_USERNAME/andromeda:latest \
  --tag ghcr.io/YOUR_GITHUB_USERNAME/andromeda:1.0.0 \
  --push \
  .

After pushing, go to your GitHub repo → Packages → set the package visibility to Public so users can pull without authentication.

⑤ HTTPS with Caddy (Recommended)

Put Caddy in front of Andromeda for automatic TLS certificates via Let's Encrypt.

Caddyfile
vault.yourdomain.com {
    reverse_proxy andromeda:3000
}
docker-compose.yml — with Caddy
version: '3.9'
services:
  andromeda:
    image: ghcr.io/YOUR_GITHUB_USERNAME/andromeda:latest
    expose: ["3000"]
    volumes:
      - andromeda-data:/data
    restart: unless-stopped

  caddy:
    image: caddy:alpine
    ports: ["80:80", "443:443"]
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy-data:/data
      - caddy-config:/config
    depends_on: [andromeda]
    restart: unless-stopped

volumes:
  andromeda-data:
  caddy-data:
  caddy-config:

Installation — For Users

Deploy Andromeda on your own machine in under 2 minutes.

🐳 Option A — Docker Run (simplest)

bash
docker run -d \
  --name andromeda \
  --restart unless-stopped \
  -p 3000:3000 \
  -v andromeda-data:/data \
  ghcr.io/YOUR_GITHUB_USERNAME/andromeda:latest

Then open http://localhost:3000 in your browser.

🐳 Option B — Docker Compose (recommended)

1
Download the compose file
curl -O https://raw.githubusercontent.com/YOUR_GITHUB_USERNAME/andromeda/main/docker-compose.yml
2
Start Andromeda
docker compose up -d
3
Open in browser
Navigate to http://localhost:3000 and follow the first-run setup wizard.

⚙ Option C — Manual (Node.js)

bash
# Requires Node.js 20+
git clone https://github.com/YOUR_GITHUB_USERNAME/andromeda.git
cd andromeda/backend
npm install
node server.js

The server serves the frontend from ../frontend/ automatically.

🚀 First Run

1
Open Andromeda
Navigate to your Andromeda URL. You'll see the welcome screen with stars.
2
Click "Enter Andromeda"
Since no vault exists yet, the setup wizard appears automatically.
3
Create a master password
Choose a strong password. This is the only key to your vault. Write it down and store it somewhere safe (a password manager, for example).
4
Add your first entry
Click + Add Entry, select a service icon, paste your recovery codes one per line, and save.
💡
On subsequent visits, clicking Enter Andromeda will show the unlock screen. Enter your master password to decrypt and access your vault.

📦 Backup & Restore

Export your vault

Click ⬇ Export in the top bar to download a .json backup. This file is encrypted — it's safe to store in cloud storage.

Backup the Docker volume

bash
# Create a backup archive
docker run --rm \
  -v andromeda-data:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/andromeda-backup-$(date +%Y%m%d).tar.gz /data

# Restore from backup
docker run --rm \
  -v andromeda-data:/data \
  -v $(pwd):/backup \
  alpine tar xzf /backup/andromeda-backup-YYYYMMDD.tar.gz -C /

Migrate to a new server

bash
# On the old server: copy the vault file
docker cp andromeda:/data/vault.enc ./vault.enc

# On the new server: restore it
docker cp ./vault.enc andromeda:/data/vault.enc

FAQ

Does Andromeda work without internet?

Yes, completely. After the container starts, everything runs locally. No external requests are made. Even the fonts are embedded.

Where is my data stored?

In a Docker named volume (andromeda-data) at /data/vault.enc inside the container. The file is always encrypted — even reading it directly from disk reveals nothing without your master password.

Can I run multiple instances?

Yes, but each instance has its own separate vault file. There is no sync between instances.

What happens if I forget my password?

Your data is gone. This is by design — Andromeda has no password recovery, no server-side key escrow, and no backdoor. Keep your master password in another password manager.

Can I use it without Docker?

Yes. Clone the repo and run node backend/server.js. Node.js 20+ required. The vault file is saved to /data/vault.enc by default — change DATA_FILE to a local path.
